<TitleProject title="Introduction" />

**Otterbots** est un framework développé par **L'Antre des Loutres** pour standardiser et accélérer le développement de nos bots Discord.
Face à la répétition du code pour chaque nouveau bot (gestion des événements, commandes, connexion à l'API...), nous avons décidé de créer une **surcouche** au-dessus de Discord.js.

C'est aujourd'hui ce framework qui propulse l'ensemble de nos bots, garantissant maintenabilité, sécurité et performance.

<TitleProject title="Fonctionnalités clés" />

Le framework encapsule des briques logiques complexes pour offrir une expérience développeur fluide :

## [Otterguard](https://github.com/L-Antre-des-Loutres/Otterbots/blob/develop/src/otterbots/docs/OTTERGUARD.md) (Modération)
Un module de modération qui a comme fonctionnalités :
- **Anti-spam / Anti-flood** intelligent.
- **Protection anti-scam** : détection de liens malveillants.
- **Filtrage de contenu** via liste blanche de domaines.

## [Otterlogs](https://github.com/L-Antre-des-Loutres/Otterbots/blob/develop/src/otterbots/docs/OTTERLOGS.md) (Journalisation)
Un système de logs puissant et granulaire :
- **7 niveaux de logs** (du `debug` au `silentlog`).
- **Webhooks Discord intégrés** pour renvoyer les erreurs critiques dans un salon staff.
- Formatage coloré dans la console pour un débogage rapide.
- Utilisation d'un module de logs spécialisé : [pino](https://github.com/pinojs/pino).

## [OtterlyAPI](https://github.com/L-Antre-des-Loutres/Otterbots/blob/develop/src/otterbots/docs/OTTERLYAPI.md) (Gestion API)
Un wrapper pour centraliser les appels API :
- Simplification des méthodes `GET` / `POST`.
- Gestion des headers et de l'authentification automatique.
- Mise en cache locale des configurations distantes.

## [Ottercache](https://github.com/L-Antre-des-Loutres/Otterbots/blob/develop/src/otterbots/docs/OTTERCACHE.md) (Persistance)
Un système de cache clé-valeur simple et efficace :
- Persistance automatique en JSON local.
- API intuitive type `Map` pour stocker des données temporaires ou durables sans base de données lourde.

## [Architecture](https://github.com/L-Antre-des-Loutres/Otterbots/blob/develop/src/otterbots/docs/OTTERBOTS_CLASS.md)

Otterbots impose une structure claire pour empêcher le "code spaghetti" :

- **Event Handler** : Chargement automatique des fichiers d'événements.
- **Command Handler** : Gestion des commandes slash avec validation des droits.
- **Scheduler** : Système de tâches planifiées (Cron jobs) intégré.

> "L'objectif était de ne plus jamais avoir à écrire `client.on('ready', ...)` à la main, mais de se concentrer sur la logique métier du bot."
